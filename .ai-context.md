## Project Overview

Primeasure Technologies Website is a comprehensive static site representing a distributor specializing in Test & Measurement, Broadcasting Solutions, and Software Development across India. The site features dynamic event management, advanced analytics tracking, and modern UI components targeting clients in electronics, broadcasting, automotive, and research industries.

## Technical Architecture

### Core Technologies
- **Frontend**: Vanilla HTML5, CSS3, ES6+ JavaScript
- **Build System**: None (static site)
- **Analytics**: Google Analytics 4 (G-SFKRS4SNFV)
- **Forms**: Google Forms integration
- **Hosting**: Static file hosting

### Key Features
1. **Dynamic Event Management**: JSON-driven events with registration system
2. **Advanced Analytics**: Custom event tracking for business metrics
3. **Responsive Design**: Mobile-first approach with touch interactions
4. **Accessibility**: ARIA attributes, keyboard navigation, screen reader support
5. **Performance**: Optimized animations, lazy loading, minimal dependencies

## File Structure & Guidelines

### HTML Structure
- **Semantic Elements**: Use proper HTML5 semantic tags (`<main>`, `<section>`, `<article>`, `<nav>`)
- **Accessibility**: Include ARIA attributes, alt text, and proper heading hierarchy
- **SEO**: Meta descriptions, structured data, proper title tags
- **Analytics**: Google Analytics tracking code on all pages

### CSS Architecture
- **Design System**: CSS custom properties in `:root` for consistent theming
- **Ant Design Inspiration**: Shadow system, border radius, transitions
- **Component-Based**: Modular styles for reusable components
- **Responsive**: Mobile-first approach with breakpoints
- **Files**: 
  - `css/style.css`: Main stylesheet with CSS variables
  - `css/responsive.css`: Mobile adaptations and media queries

### JavaScript Modules
- **ES6+ Classes**: Modern JavaScript with class-based architecture
- **Modular Design**: Each file handles specific functionality
- **Error Handling**: Comprehensive error handling and fallbacks
- **Performance**: RequestAnimationFrame, efficient event delegation

#### Core JavaScript Files:
1. **`main.js`**: Scroll animations, navbar behavior, Intersection Observer
2. **`carousel.js`**: Advanced carousel system with accessibility
3. **`menu.js`**: Mobile navigation, dropdown menus
4. **`analytics.js`**: Google Analytics enhanced tracking
5. **`EventManager.js`**: Dynamic event loading and management
6. **`EventRouter.js`**: Hash-based routing for events
7. **`FormValidator.js`**: Real-time form validation
8. **`SmoothInfiniteScroll.js`**: Customer logos animation

### Asset Organization
```
images/
```

### Data Management
- **`data/events.json`**: Centralized event database
- **Fallback System**: Hardcoded data in EventManager.js for offline functionality
- **Dynamic Loading**: Fetch API with error handling

## Development Guidelines

### Code Style
- **JavaScript**: ES6+ features, arrow functions, async/await
- **CSS**: BEM-like naming, CSS custom properties
- **HTML**: Semantic structure, proper indentation
- **Accessibility**: WCAG 2.1 compliance focus

### Adding New Content

#### New Pages
1. Follow existing HTML structure and navigation patterns
2. Include Google Analytics tracking code
3. Update navigation in all relevant pages
4. Maintain responsive design principles
5. Add to sitemap.xml for SEO

#### New Events
1. Update `data/events.json` with event details
2. Add event images to `images/events/`
3. Test registration flow if enabled
4. Update fallback data in EventManager.js

#### New Customer Logos
1. Add to `images/customers/` directory
2. Optimize for web (150x75px recommended)
3. Update customer carousel in relevant pages
4. Ensure proper alt text for accessibility

### Performance Optimization
- **Images**: Optimize for web, consider WebP format
- **CSS**: Use CSS custom properties, avoid unnecessary specificity
- **JavaScript**: Event delegation, proper cleanup, minimal DOM manipulation
- **Loading**: Local server required for JSON data (CORS)

### SEO & Analytics
- **Meta Tags**: Unique title and description per page
- **Structured Data**: Business schema markup
- **Analytics Events**: Track user interactions automatically
- **Sitemap**: Keep updated with new pages
- **Privacy**: GDPR-compliant privacy policy
